##　branchとは
Git を用いて開発を行う際には「branch」を活用する．
branch を用いると履歴の流れを分岐して管理することができる．分岐させた branch は他の branch の影響を受けないため，同じリポジトリ中で複数の変更を同時に進めていくことができる．

# まず main branch を用意する（リポジトリ作成時に自動で作成される）．

# develop branch を作成し，main branch から切り替える．

# develop branch でコードを書き，開発を進める．

# 区切りがついたらコミットを実行し，push する．

# main branch に対して「プルリクエスト」を作成する．

# 作成されたプルリクエストを「マージ」し，develop branch で開発した内容を「main」branch に反映させる．
